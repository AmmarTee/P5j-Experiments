# P5.js Physics Simulations -- Agent Instructions

## Overview

This is a monorepo (with git submodules) for browser-based physics simulations built with p5.js. Each simulation is a standalone project with its own GitHub repository, linked here as a submodule. The simulations are produced for a YouTube channel focused on physics, chaos theory, and mathematical visualization.

## Repository Architecture

```
P5j-Experiments/               (parent repo: AmmarTee/P5j-Experiments)
  .agent.md                    Agent instructions (this file)
  .gitignore                   Excludes recordings, final_videos, produce_video.py
  .gitmodules                  Submodule definitions
  index.html                   Dashboard -- links to all simulations
  server.py                    Shared dev server (port 8001)
  produce_video.py             Recording/editing automation (git-ignored, local only)
  3BodyProblem/                Submodule -> AmmarTee/3BodyProblem
  DoublePendulum/              Submodule -> AmmarTee/DoublePendulum
  ...
```

Each simulation submodule contains:

```
SimulationName/
  index.html                   Entry point, loads p5.js 1.9.0 from CDN
  sketch.js                    All simulation, rendering, audio, and UI code
```

## Branding and Visual Standards

- Background: `background(5, 5, 15)` -- near-black with slight blue tint
- Palette: Red, blue, green glowing particles on dark backgrounds
- Glow effect: Layered concentric ellipses with decreasing alpha
- Trails: Color-matched, alpha-faded, stroke-weight-ramped polylines
- Font: Monospace for all HUD text
- Grid: Subtle white lines at alpha ~20-25, toggleable
- HUD: Top-left aligned, showing title, preset, time, body parameters, energy, and controls

## Physics Standards

All simulations must use physically accurate models:

- **Integration**: RK4 (preferred for chaotic systems) or Velocity-Verlet (for N-body). Euler methods are not acceptable.
- **Sub-stepping**: 8-16 sub-steps per frame minimum. Higher for stiff or chaotic systems.
- **Units**: Use SI units internally (meters, kilograms, seconds, rad/s, Joules). Convert to pixels only for rendering via a `PX_PER_M` constant.
- **Constants**: Use real physical constants (g = 9.81 m/s^2, G = 6.674e-11 for gravity where applicable). Tuned constants are acceptable only for N-body simulations where real G produces imperceptible motion at display scale.
- **Softening/clamping**: Required for N-body to prevent singularities. Not needed for constrained systems (pendulums, springs).
- **Conservation**: Display total energy (KE + PE) in the HUD. Energy drift over time indicates integration quality.
- **Timestep**: Sync to real time where possible (`DT = 1/60` for 60fps). For N-body where real-time is impractical, use a tuned DT with clear documentation.

## Audio Standards (Web Audio API)

All audio is procedural. No external audio files.

- **Ambient drone**: 3-4 detuned sawtooth oscillators through a low-pass filter (~160Hz cutoff), modulated by a slow LFO (0.05-0.08 Hz). Master drone gain ~0.06-0.08.
- **Reactive oscillators**: Map simulation parameters to audio:
  - Velocity/angular velocity -> oscillator pitch and gain
  - Proximity/distance -> sub-bass rumble intensity
  - Acceleration changes/chaos -> filter cutoff and resonance
- **Impact SFX**: On preset switch or reset -- sine sweep (80Hz -> 25Hz over 0.4s) layered with a short noise burst through a low-pass filter.
- **Autoplay compliance**: Audio context must be created on first user interaction (click or keypress), never on page load.
- **Master gain**: 0.35 default. Toggle mute with M key (set gain to 0, not disconnect).

## Controls (Mandatory for all simulations)

| Key       | Action              |
|-----------|---------------------|
| SPACE     | Pause / Resume      |
| R         | Reset current preset|
| 1-5       | Switch preset       |
| T         | Toggle trails       |
| G         | Toggle grid         |
| M         | Toggle sound        |
| + / -     | Speed up / slow down (0.25x to 4x) |
| V         | Start/stop recording|

## Recording

Each simulation includes a built-in MediaRecorder that captures the canvas at 60fps as webm. The `V` key toggles recording. A pulsing red "REC" indicator appears in the top-right corner during recording. The file auto-downloads on stop.

The `produce_video.py` script (git-ignored) automates batch recording via Playwright and editing via ffmpeg. It is not part of the public repos.

## Presets

Each simulation must include 3-5 presets switchable with number keys:

- Each preset demonstrates a distinct physical behavior or phenomenon
- Presets are named descriptively (e.g., "Classic Chaos", "Figure-Eight", "Near Flip")
- Preset data is stored in a `PRESETS` array of plain objects
- `loadPreset(index)` resets all state, reinitializes the simulation, resets the camera, and plays the impact SFX

## Camera

- Smooth lerp-based follow (factor ~0.03) tracking the center of mass or center of action
- Auto-zoom to keep all elements visible with padding
- Zoom constrained to a reasonable range (e.g., 0.15-3.0)
- Camera state: `{ x, y, zoom, targetZoom }`

## Code Conventions

- `pixelDensity(1)` in setup for consistent cross-device performance
- `windowResized()` must call `resizeCanvas(windowWidth, windowHeight)`
- No external dependencies beyond the p5.js CDN (v1.9.0)
- No build tools, no bundlers, no npm -- plain JS served over HTTP
- HUD draws after `resetMatrix()` to render in screen space
- Use `const` for immutable values, `let` for state. Avoid `var`.
- Descriptive section headers with `// ====` comment blocks

## Adding a New Simulation

1. Create folder: `E:\Personal-Projects\P5j-Experiments\SimulationName\`
2. Create `index.html` from the standard template (see existing simulations)
3. Create `sketch.js` following all standards above
4. Test locally: `python server.py` from the parent directory, navigate to `SimulationName/index.html`
5. Verify: all presets work, audio plays after interaction, controls function, no console errors, 60fps performance, energy conservation is reasonable
6. Initialize git, create GitHub repo, push:
   ```powershell
   cd E:\Personal-Projects\P5j-Experiments\SimulationName
   git init
   git add .
   git commit -m "Initial commit: SimulationName simulation"
   gh repo create AmmarTee/SimulationName --public --source=. --remote=origin --push
   ```
7. Add as submodule to parent:
   ```powershell
   cd E:\Personal-Projects\P5j-Experiments
   git submodule add https://github.com/AmmarTee/SimulationName.git SimulationName
   ```
8. Update `index.html` dashboard with a new card for the simulation
9. Commit and push parent repo:
   ```powershell
   git add .
   git commit -m "Add SimulationName submodule"
   git push origin master
   ```

## Updating an Existing Simulation

1. Make changes inside the submodule directory
2. Commit and push the submodule:
   ```powershell
   cd E:\Personal-Projects\P5j-Experiments\SimulationName
   git add .
   git commit -m "Description of changes"
   git push origin main
   ```
3. Update the parent repo to track the new submodule commit:
   ```powershell
   cd E:\Personal-Projects\P5j-Experiments
   git add SimulationName
   git commit -m "Update SimulationName submodule"
   git push origin master
   ```
