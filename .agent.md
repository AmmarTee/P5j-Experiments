# P5.js Physics Simulation YouTube Channel Agent

## Purpose

Create visually stunning, YouTube-ready physics simulations using p5.js for a channel focused on physics, chaos theory, and mathematical visualization.

## Channel Branding

- Color palette: Red, Blue, Green glowing particles on pure black backgrounds
- Aesthetic: Cinematic, neon glow, high contrast, dark space theme
- Font: Monospace for HUD overlays
- Audio: Procedural Web Audio API (no external audio files)
- GitHub: All projects are open-source and linked in video descriptions

Each simulation lives in its own folder under `E:\Personal-Projects\P5j-Experiments\`:

```

  .agent.md          (this file)
  3BodyProblem/

  ...
```

Each project folder MUST contain:

```
ProjectName/
  index.html      - Entry point, loads p5.js from CDN and sketch.js
  sketch.js       - All simulation code
  server.py       - Python HTTP server to run locally
```

## How to Build a Simulation

### Step 1: Create the project folder

Create a new folder under `E:\Personal-Projects\P5j-Experiments\` named after the simulation (PascalCase, no spaces).

### Step 2: index.html

```html
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>SIMULATION_NAME</title>
  <script src="https://cdnjs.cloudflare.com/ajax/libs/p5.js/1.9.0/p5.min.js"></script>
  <style>
    * { margin: 0; padding: 0; box-sizing: border-box; }
    body { background: #000; overflow: hidden; display: flex; justify-content: center; align-items: center; height: 100vh; }
    canvas { display: block; }
  </style>
</head>
<body>
  <script src="sketch.js"></script>
</body>
</html>
```

### Step 3: sketch.js requirements

Every simulation MUST include:

**Visuals:**
- Full-screen canvas using `windowWidth` and `windowHeight`
- Dark background (`background(5, 5, 15)`)
- Glowing bodies/particles with layered alpha ellipses for glow effect
- Colored trails that fade over time
- Smooth camera that auto-follows and auto-zooms to keep all elements visible
- Subtle background grid (toggleable with G key)
- Clean HUD overlay showing: simulation title, current config/preset name, time elapsed, key parameters, and keyboard controls

**Physics:**
- Use accurate numerical integration (Velocity-Verlet preferred, RK4 acceptable)
- Use sub-stepping (4-8 sub-steps per frame) for stability
- Include softening or clamping to prevent numerical explosions
- Display energy or other conservation metrics in the HUD

**Presets:**
- Include 3-5 interesting presets/configurations switchable with number keys 1-5
- Each preset should demonstrate a different behavior or phenomenon
- Name each preset descriptively

**Audio (Web Audio API, procedural only):**
- Ambient drone: layered detuned oscillators through low-pass filter with slow LFO
- Reactive sounds: map simulation parameters (velocity, proximity, energy) to oscillator pitch, volume, and filter cutoff
- Impact/transition sound on preset switch
- Audio starts on first user interaction (click or keypress) to comply with browser autoplay policy
- Master gain node for volume control
- Toggle sound with M key

**Controls (standard across all simulations):**
- SPACE: Pause/Resume
- R: Reset current preset
- 1-5: Switch preset
- T: Toggle trails
- G: Toggle grid
- M: Toggle sound

**Code quality:**
- Use `pixelDensity(1)` for consistent performance
- Handle `windowResized()` to resize canvas
- No external dependencies beyond p5.js CDN
- All audio is procedural via Web Audio API, no audio files

### Step 4: server.py

```python
import http.server
import webbrowser
import os

PORT = 8000
os.chdir(os.path.dirname(os.path.abspath(__file__)))

webbrowser.open(f"http://localhost:{PORT}")
print(f"Serving at http://localhost:{PORT}  (Ctrl+C to stop)")
http.server.HTTPServer(("", PORT), http.server.SimpleHTTPRequestHandler).serve_forever()
```

### Step 5: Test locally

Run `python server.py` from the project folder and verify:
- All presets work and look visually interesting
- Audio plays after first interaction
- All keyboard controls function
- No console errors
- Performance is smooth at 60fps on a standard machine
- Auto-zoom and camera tracking work correctly

### Step 6: Push to GitHub

After the simulation is complete and tested:

1. Initialize git in the project folder if not already done
2. Check if a public repo exists on GitHub under the owner's account with the project name
3. If the repo does NOT exist, create a new **public** repository on GitHub
4. Add the remote, commit all files, and push to the main branch

Use these commands:
```powershell
cd E:\Personal-Projects\P5j-Experiments\ProjectName
git init
git add .
git commit -m "Initial commit: ProjectName simulation"
gh repo create AmmarTee/ProjectName --public --source=. --remote=origin --push
```

If the repo already exists:
```powershell
git add .
git commit -m "Update: description of changes"
git push origin main
```
